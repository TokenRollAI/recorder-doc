# 快速上手

让我们在 **5分钟内了解 recorder**。

## 前置

### 下载uvx

> 参考 https://docs.astral.sh/uv/getting-started/installation/

<Tabs>
<TabItem value="mac-linux" label="macOS & Linux">

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

如果没有安装`curl`的话, 可以使用`wget`

```bash
wget -qO- https://astral.sh/uv/install.sh | sh
```

</TabItem>
<TabItem value="windows" label="Windows">

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

</TabItem>
</Tabs>

### 配置SPEC-mcp

> 项目地址: https://github.com/TokenRollAI/SPEC-mcp

在你使用的AI Ide的mcp配置中添加:
```
{
    "mcpServers": {
        "spec-mcp": {
            "command": "uvx",
            "args": [
                "mcp-spec@latest"
            ]
        }
    }
}
```

因为recorder使用了vscode的相当高的权限, 例如获取terminal的输入的命令以及命令的输出.

如果您使用的是cursor/windsurf/trae, 默认使用的就是open-vsx

### 注意!!!

1. recorder依赖git的使用, 所以您必须保证所在的项目是通过git进行版本管理
2. 如果想要得到最好的效果, 建议在一个全新的的,没有任何stash change的分支进行录制, 这样可以得到最干净的改动

### 生成SOP-SPEC

打开你使用的AI IDE的chat框, 然后选择一个有写权限的模式(例如Cursor/AugmentCode的Agent模式)

然后输入: `call generate_operation_md`

## Just Do Once

在完成了以上所有的操作之后, 恭喜你脱离苦海!

后续所有的类似操作, 例如: 增加接口, 增加组件, 增加SQL等等, 您都可以直接要求LLM参考这个md文件.

在常规的AI IDE中, 你只需要引用此文件(在Cursor/AugmentCode中, 您只需要`@`此文件), 然后说出您的需求, AI会参考您之前的操作, 进行高效的开发!

我们建议的组织形式类似于:

```
read and follow the SOP in the md.

now, i want to [your new requirements]
```
